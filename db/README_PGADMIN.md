# PostgreSQL + pgAdmin: пошагово с нуля

Эта инструкция под твой проект и новые "чистые" SQL-пакеты:

- `db/create_db.sql`
- `db/schema.sql`
- `db/seed.sql`
- `db/seed_more.sql`
- `db/verify.sql`

## 0) Что будет в итоге

Ты поднимешь новую локальную БД `pc_salon`, создашь структуру таблиц, добавишь стартовые данные и проверишь, что API может с ней работать.

---

## 1) Открыть pgAdmin и подключиться

1. Запусти `pgAdmin 4`.
2. Слева найди `Servers`.
3. Открой свой сервер `PostgreSQL 17` (или похожее имя).
4. Если запросит пароль — введи пароль пользователя `postgres`.

---

## 2) Создать БД (скриптом)

### Важно:
`create_db.sql` нужно запускать, когда выбрана БД `postgres` (системная), а не `pc_salon`.

1. В дереве слева нажми на `Databases` -> `postgres`.
2. ПКМ -> `Query Tool`.
3. Открой файл `db/create_db.sql`.
4. Нажми ▶ `Execute`.

После этого в `Databases` появится `pc_salon` (иногда нужно нажать Refresh).

---

## 3) Создать таблицы и индексы

1. Выбери БД `pc_salon`.
2. ПКМ -> `Query Tool`.
3. Открой `db/schema.sql`.
4. Нажми ▶ `Execute`.

Что создается:

- таблицы: `users`, `categories`, `products`, `orders`, `order_items`, `reviews`
- внешние ключи (связи)
- индексы для ускорения частых запросов
- триггеры `updated_at`

---

## 4) Заполнить базовыми данными

1. В том же Query Tool открой `db/seed.sql`.
2. Нажми ▶ `Execute`.

Это добавит:

- 3 категории
- несколько тестовых товаров

---

## 5) Проверить корректность

1. Открой `db/verify.sql`.
2. Нажми ▶ `Execute`.

Ожидаемо:

- список таблиц отображается
- в `categories/products` есть строки
- проверки "broken_*" должны быть `0`

---

## Дополнительно: наполнить демо-данными

Если хочешь более "живой" интерфейс для демонстрации диплома:

1. Открой `db/seed_more.sql`
2. Нажми ▶ `Execute`

Скрипт добавит:

- дополнительные товары в каталог
- тестовых пользователей
- тестовые заказы и позиции заказов
- опубликованный тестовый отзыв

---

## 6) Где смотреть таблицы, столбцы, связи и данные в pgAdmin

### Таблицы

- `Databases -> pc_salon -> Schemas -> public -> Tables`

### Столбцы таблицы

- выбери таблицу, например `products`
- раскрой `Columns`

### Связи (foreign keys)

- выбери таблицу
- раскрой `Constraints -> Foreign Keys`

### Данные таблицы

- ПКМ по таблице -> `View/Edit Data -> All Rows`

## 7) Подключение текущего FastAPI к этой БД

Проверь файл `python_api/.env`:

```env
DB_HOST=localhost
DB_PORT=5432
DB_NAME=pc_salon
DB_USER=postgres
DB_PASS=11111111
ALLOWED_ORIGINS=*
```

Запуск API:

```powershell
cd C:\xampp\htdocs\pc_salon
.\run_local_api.ps1
```

Проверка:

- `http://127.0.0.1:8000/api/health`
- `http://127.0.0.1:8000/api/categories`

---

## 8) Почему эта схема "лучше и производительнее"

1. **Индексы под реальные запросы**:
   - товары по категории и активности
   - заказы по пользователю/дате/статусу
   - отзывы по продукту/публикации/дате
2. **Ограничения данных**:
   - `CHECK` на рейтинг, статусы, суммы, количество
   - снижает мусорные данные
3. **Целостность связей**:
   - внешние ключи не дают "битые" ссылки
4. **Триггеры `updated_at`**:
   - автоматическое обновление времени изменения записи

---

## 9) Если что-то не выполняется

- Проверь, что запускаешь файл в правильной БД:
  - `create_db.sql` -> в `postgres`
  - остальные -> в `pc_salon`
- Внизу вкладки `Messages` всегда видно конкретную SQL ошибку.

---

## 10) Автономный локальный режим (мини-приложение без Telegram)

Цепочка: **браузер -> FastAPI (`python_api`) -> PostgreSQL**.

1. Подними БД (скрипты выше) и при необходимости выполни `seed.sql` / `seed_more.sql`.
2. Запусти API:
   - `C:\xampp\htdocs\pc_salon\run_local_api.ps1`
3. Открой сайт:
   - `http://127.0.0.1:8000/`

В коде включён режим **только база** (`AURUM_DB_ONLY`): каталог и поиск не подмешивают мок-данные, если API недоступен — показывается подсказка. Для старого офлайн-режима в консоли: `window.AURUM_DB_ONLY = false` (до загрузки модульного скрипта это задаётся в отдельной вкладке сложно; проще временно править `index.html`).

Запись в БД с сайта идёт через API: заказы (`POST /api/orders`), пользователи (`POST /api/users`), отзывы (`POST /api/reviews`).

**Примечание:** админка — `admin/products.html` и `admin/orders.html`; они вызывают тот же FastAPI (`/api/products`, `/api/orders`). Открывать через:

- `http://127.0.0.1:8000/admin/products.html`
- `http://127.0.0.1:8000/admin/orders.html`
